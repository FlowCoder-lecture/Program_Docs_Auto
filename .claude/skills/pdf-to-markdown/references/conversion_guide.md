# PDF to Markdown 상세 변환 가이드

## 문서 유형별 변환 패턴

### 1. 정부지원사업 신청서

정부지원사업 신청서는 보통 다음 구조를 가짐:

```markdown
---
title: "[사업명] 신청서"
type: "정부지원사업 신청서"
source: "파일명.pdf"
agency: "주관기관명"
deadline: "YYYY-MM-DD"
---

# [사업명] 신청서

## 1. 신청기업 현황
### 1.1 기업 기본정보
### 1.2 대표자 정보

### 2.1 과제명
### 2.2 기술 분야
### 2.3 개발 목표

## 3. 기술개발 내용
### 3.1 기술개발 필요성
### 3.2 기술개발 내용
### 3.3 개발 방법론

## 4. 추진체계
### 4.1 추진 일정
### 4.2 참여인력
### 4.3 외부 협력

## 5. 사업화 계획
### 5.1 목표 시장
### 5.2 사업화 전략
### 5.3 매출 계획

## 6. 소요예산
### 6.1 총괄 예산
### 6.2 세부 항목

## 7. 첨부서류
```

## 표 변환 상세 가이드

### 단순 표
```
원본 PDF:
┌────────┬────────┬────────┐
│ 항목   │ 내용   │ 비고   │
├────────┼────────┼────────┤
│ A      │ B      │ C      │
└────────┴────────┴────────┘

마크다운:
| 항목 | 내용 | 비고 |
|------|------|------|
| A | B | C |
```

### 입력 필드가 있는 표
```
원본 PDF:
┌────────────┬──────────────────┐
│ 회사명     │                  │
├────────────┼──────────────────┤
│ 대표자     │                  │
└────────────┴──────────────────┘

마크다운:
| 항목 | 내용 |
|------|------|
| 회사명 | **[입력: 회사명]** |
| 대표자 | **[입력: 대표자명]** |
```

### 병합 셀이 있는 표
```
원본 PDF:
┌────────┬────────┬────────┐
│ 인건비          │ 금액   │
├────────┼────────┼────────┤
│        │ 급여   │ 1,000  │
│        │ 상여   │ 500    │
└────────┴────────┴────────┘

마크다운:
| 대분류 | 소분류 | 금액 |
|--------|--------|------|
| **인건비** | | |
| | 급여 | 1,000 |
| | 상여 | 500 |
```

### 다단 표 (복잡한 구조)
```
원본 PDF: 3열 이상의 복잡한 표

마크다운 (분리 권장):
#### 기본 정보
| 항목 | 내용 |
|------|------|
| ... | ... |

#### 상세 정보
| 항목 | 내용 |
|------|------|
| ... | ... |
```

## 입력 필드 유형별 표현

### 텍스트 입력
```markdown
**[입력: 필드명]**
**[입력: 회사명을 입력하세요]**
```

### 장문 텍스트
```markdown
**[입력 - 여러 줄: 사업 내용을 상세히 기술하세요 (500자 이내)]**
```

### 숫자/금액
```markdown
**[금액: 원]**
**[숫자: 개]**
**[비율: %]**
```

### 날짜
```markdown
**[날짜: YYYY-MM-DD]**
**[기간: YYYY-MM-DD ~ YYYY-MM-DD]**
```

### 선택형
```markdown
**[선택: 예 / 아니오]**
**[선택: IT/SW / 제조 / 서비스 / 바이오 / 기타]**
```

### 체크박스
```markdown
- [ ] 항목 1
- [ ] 항목 2
- [ ] 항목 3
```

### 서명
```markdown
**[서명]**
**[직인]**
```

## 특수 요소 처리

### 페이지 번호
- 페이지 번호는 변환하지 않음
- 필요시 HTML 주석으로 표시: `<!-- Page 3 -->`

### 머리글/바닥글
- 반복되는 머리글/바닥글은 생략
- 중요한 정보(문서번호 등)는 메타데이터에 포함

### 워터마크
- 워터마크는 변환하지 않음
- 필요시 메타데이터에 표시

### 이미지
```markdown
> 🖼️ **[이미지: 설명]**
> - 위치: 원본 N페이지
> - 파일: image_name.png (추출시)
```

### 도표/차트
```markdown
> 📊 **[차트: 차트 제목]**
> - 유형: 막대그래프 / 파이차트 / 선그래프
> - 데이터 요약:
>   - 항목1: 값1
>   - 항목2: 값2
```

### QR코드/바코드
```markdown
> 🔗 **[QR코드]**
> - URL: https://example.com (해독시)
```

## 품질 검증 체크리스트

### 구조적 완전성
- [ ] 원본의 모든 섹션이 포함됨
- [ ] 제목 계층(H1~H4)이 논리적임
- [ ] 목차와 본문 섹션이 일치함

### 표 정확성
- [ ] 모든 표가 올바르게 변환됨
- [ ] 열 정렬이 맞음
- [ ] 병합 셀이 적절히 처리됨

### 입력 필드
- [ ] 모든 입력 필드가 **[입력:]** 형식으로 표시됨
- [ ] 필드 설명이 명확함
- [ ] 필수/선택 여부가 구분됨

### 마크다운 문법
- [ ] 문법 오류 없음
- [ ] 들여쓰기 일관성
- [ ] 빈 줄 처리 적절함

### 가독성
- [ ] 섹션 간 구분이 명확함
- [ ] 표가 깨지지 않음
- [ ] 긴 텍스트 줄바꿈 적절함
